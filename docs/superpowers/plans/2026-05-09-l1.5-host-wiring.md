# L1.5a: Host Linux Wiring Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Refactor `crates/host/src/` so the Windows-coupled guts move into `crates/host/src/platform/win.rs`, add a thin `crates/host/src/platform/linux.rs` that consumes the L1 platform crates, expose a cfg-transparent `platform::*` API, and edit `crates/host/src/lib.rs` so `prdt host` builds and starts on WSL2 Linux end-to-end (UDP listen → handshake → capture loop via `LinuxSwProducer` → input/clipboard sync via `prdt-input-linux`). No viewer wiring (deferred to L1.5b).

**Architecture:** Approach 2 (factory-centric). `VideoEncoderBackend` enum closes inside `platform/win/`. `lib.rs` calls a single `platform::build_video_producer(...)` factory whose return value is `Box<dyn VideoProducer>`. The `prdt_protocol::VideoProducer` trait is extended with one method, `fn backend_name() -> &'static str`, so the factory returns just the box (no tuple workaround). Linux smoke is verified via an `#[ignore]`-gated tokio integration test that drives a fake-viewer transport against `prdt_host::run_with_args`.

**Tech Stack:** Rust 1.85+, Cargo workspace, existing crates `prdt-host`, `prdt-protocol`, `prdt-media-win`, `prdt-media-sw`, `prdt-input-win`, `prdt-media-linux` (L1), `prdt-input-linux` (L1). No new external deps.

**Spec:** `docs/superpowers/specs/2026-05-09-l1.5-host-wiring-design.md` (commit `b20def0`).

**Branch:** `phase-l1-linux-poc` (already created during L1; this plan continues on it). No new branch needed.

**Regression bar:** `cargo check --workspace` and `cargo test --workspace --exclude prdt-latency-bench` MUST stay green on Linux native (the WSL2 dev env's default target). `cargo check --workspace --target x86_64-unknown-linux-gnu` MUST land newly green for `prdt-host`. `cargo clippy --workspace --target x86_64-unknown-linux-gnu --all-targets -- -D warnings` MUST stay green. Existing tests in `crates/host/`, `crates/protocol/`, `crates/media-win/`, `crates/media-linux/`, `crates/input-linux/` MUST continue to pass.

/// Per-OS opaque output descriptor. Windows holds the real `OutputInfo`;

pub type OutputDescriptor = OutputInfo;

/// Pick the default output (monitor) for capture. Internally enumerates
/// adapters + outputs.
pub fn pick_default_output(_args: &crate::Args) -> anyhow::Result<OutputDescriptor> {
    let adapter = pick_default_adapter().context("pick_default_adapter")?;
    let outputs = enumerate_outputs_for_adapter(&adapter).context("enumerate_outputs_for_adapter")?;
    let primary = outputs
        .into_iter()
        .next()
        .ok_or_else(|| anyhow::anyhow!("no DXGI output found on adapter"))?;
    Ok(primary)
}

/// Build a boxed `VideoProducer` for the requested encoder backend.
/// `args_encoder` is the raw CLI string ("auto" | "nvenc" | "mf" | "openh264");
/// `negotiated_codec` is the codec the viewer agreed to in HelloAck.
pub fn build_video_producer(
    args_encoder: &str,
    output: &OutputDescriptor,
    bitrate_bps: u32,
    fps: u32,
    negotiated_codec: prdt_protocol::Codec,
) -> anyhow::Result<Box<dyn VideoProducer>> {
    // Reuse the existing pick_encoder + match-on-VideoEncoderBackend logic
    // from the legacy lib.rs. To avoid duplicating ~80 lines, this fn
    // delegates to the routed helpers below. They are private to win.rs.
    let dev = D3d11Device::create(&pick_default_adapter().context("adapter")?)
        .context("D3D11 device")?;
    let width = (output.desktop_rect.right - output.desktop_rect.left) as u32;
    let height = (output.desktop_rect.bottom - output.desktop_rect.top) as u32;
    let backend = pick_encoder(args_encoder, &dev, width, height, bitrate_bps, negotiated_codec)
        .context("pick_encoder")?;
    let producer: Box<dyn VideoProducer> = match backend {
        VideoEncoderBackend::Hw(enc) => Box::new(
            DxgiNvencProducer::with_encoder(&dev, output, enc).context("hw producer")?,
        ),
        VideoEncoderBackend::SwH264(enc) => Box::new(
            DxgiSwProducer::with_encoder(&dev, output, *enc).context("sw producer")?,
        ),
    };
    let _ = fps; // fps is currently encoded into the encoder cfg upstream; flagged unused here
    Ok(producer)
