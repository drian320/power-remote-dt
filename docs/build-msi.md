# Building the MSI installer

The Phase 4 G4 MSI installer is generated by [`cargo-wix`](https://github.com/volks73/cargo-wix), which drives [WiX Toolset](https://wixtoolset.org/) 3.14.

## One-time setup

1. Install WiX Toolset 3.14:
   - Download `wix314.exe` from <https://github.com/wixtoolset/wix3/releases/tag/wix3141rtm>
   - Run as Administrator. Default install path: `C:\Program Files (x86)\WiX Toolset v3.14\bin`
   - Add that bin directory to PATH (so `cargo-wix` can find `candle.exe` / `light.exe`).
2. Install the cargo subcommand:
   ```bash
   cargo install cargo-wix --version 0.3.9
   ```
3. (One-time) Generate the icon:
   ```bash
   cargo run -p prdt-gui-host --bin mkicon
   ```
   This converts `crates/gui-host/assets/tray-listening.png` (G3 build artifact) into a multi-resolution `crates/gui-host/resources/prdt-icon.ico`.

## Building a release MSI

```bash
# 1. Build all binaries with the icon resource embedded.
cargo build --release -p prdt-host -p prdt-viewer -p prdt-viewer-overlay

# 2. Generate the MSI.
cargo wix --no-build --nocapture
```

Output: `target/wix/prdt-setup-vX.Y.Z.msi` (where `X.Y.Z` is the workspace version).

`--no-build` tells `cargo-wix` to use the binaries we already produced in step 1. Without it, `cargo-wix` runs `cargo build --release` itself, which is fine but slower in CI loops.

## Smoke test on a clean Win10/11 VM

1. Copy `target/wix/prdt-setup-vX.Y.Z.msi` to a Windows 10/11 VM.
2. Double-click. Accept the license dialog. Click Install.
3. Confirm the install goes to `%LOCALAPPDATA%\prdt\bin\` (no UAC prompt).
4. Open the Start menu → "Power Remote Desktop" group → click Host. Expect the GUI to open.
5. Right-click the tray icon (Phase 4 G3) → Quit.
6. Re-launch via Start menu. Confirm the saved config (`host-key.bin`, `host-id.txt`) carries over to the second launch.
7. Uninstall via "Apps & features". Confirm `%LOCALAPPDATA%\prdt\bin\` is gone but `%APPDATA%\prdt\config.toml` persists (per spec — user data is preserved).

## Releasing a new version

1. Bump `version` in the workspace `Cargo.toml`.
2. Build the MSI (above).
3. `git tag -a vX.Y.Z` matching the workspace version.
4. Push the tag. `gh release create vX.Y.Z target/wix/prdt-setup-vX.Y.Z.msi` (or use the GitHub web UI).
5. The next time a user's installed `prdt-host.exe` runs, the auto-update check will surface the new version (after the 7-day cooldown, or immediately via Settings → Check for updates).

## Troubleshooting

- **`candle.exe` not found** — WiX 3.14 PATH is wrong. Add `C:\Program Files (x86)\WiX Toolset v3.14\bin` and reopen the shell.
- **`prdt-icon.ico missing` warning during build** — Run the `mkicon` step above, then rebuild.
- **MSI installs but Start Menu shortcuts are broken** — Rebuild with the latest `wix/main.wxs`; the `[APPLICATIONFOLDER]` resolution requires WiX 3.14.
- **`UpgradeCode` mismatch** — Never change `wix/main.wxs`'s `UpgradeCode` GUID. If you regenerate it, existing installs become unrecognized and can't be uninstalled by the new MSI; users must manually uninstall the old version first.
- **Auto-update check fails with "rate limit"** — GitHub limits unauthenticated API calls to 60/hour. Wait, or set `HTTPS_PROXY` if behind an enterprise proxy that has higher quotas.
